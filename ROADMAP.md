# AI Планер дня — Roadmap

## Продукт

Mobile-first веб-застосунок: юзер вивалює хаотичний текст думок, AI перетворює його на задачі,
юзер перевіряє й підтверджує, а потім AI за один тап складає реалістичний план на сьогодні.

## Ключові рішення (MVP)

- **Mobile-first з першого екрана** — не окремий етап полірування. Кожен екран одразу верстається
  під телефон (великі тач-елементи, все зручно однією рукою), десктоп — адаптація поверх мобільної
  версії, а не навпаки.
- **Без логіну** — один користувач, без реєстрації/акаунтів.
- **Зберігання задач — localStorage у браузері**, не серверна база даних. Свідомий компроміс заради
  швидкості MVP: задачі живуть на одному пристрої/браузері. Шар зберігання проєктується так, щоб
  пізніше замінити на серверну БД без переписування застосунку.
- **AI-виклик завжди через сервер** (Next.js API route). Ключ `ANTHROPIC_API_KEY` — тільки в змінних
  середовища (Vercel / локальний `.env.local`), ніколи не потрапляє в браузер.
- **Робочі години для плану дня** — поля `day_start` / `day_end`, за замовчуванням 09:00–18:00,
  редаговані юзером перед генерацією плану.
- **Позначка "виконано"** (галочка) — базова функція з першого дня, не відкладається.

## Схема задачі

| Поле | Що це | Навіщо потрібне |
|---|---|---|
| `id` | Унікальний номер задачі | Точно вказати "оцю задачу редагувати/видалити" |
| `title` | Назва задачі | Головне, що бачить юзер у списку |
| `source_text` | Шматок оригінального тексту, з якого AI зробив цю задачу | Звірити з оригіналом на екрані прев'ю, якщо AI щось не так зрозумів |
| `priority` | Високий / середній / низький / не вказано | AI проставляє автоматично; потрібно для побудови плану дня |
| `estimated_minutes` | Орієнтовний час у хвилинах | Без цього AI не збудує реалістичний план дня |
| `deadline` | Дата дедлайну (необов'язково) | Показати терміновість AI і юзеру |
| `status` | inbox / today / done | Де "живе" задача зараз |
| `scheduled_date` | Дата, на яку заплановано (якщо status = today) | Відрізнити план на сьогодні від загального списку |
| `position` | Порядковий номер у плані дня | Порядок задач, який визначив AI |
| `created_at` | Коли задачу створено | Сортування за замовчуванням, "вік" задачі в Inbox |
| `completed_at` | Коли позначено виконаною (необов'язково) | Галочка "виконано" |

Налаштування дня (не задача, а глобальні налаштування): `day_start`, `day_end` (за замовчуванням 09:00–18:00).

## Екрани

- **Welcome** (тільки при першому вході) — одне речення цінності + кнопка "Почати".
- **Capture** — "Що в голові?": велике текстове поле, кнопка "Розібрати" знизу. Порожній стан:
  приклад-текст + кнопка "Спробувати з прикладом".
- **Preview** — картки розпарсених задач (назва, пріоритет, час, дедлайн — все редагується),
  хрестик — видалити, кнопка "Зберегти всі" знизу.
- **Inbox** — список усіх задач із галочкою "виконано". Порожній стан із підказкою й кнопкою на Capture.
- **Today** — поля робочих годин + кнопка "Сформувати план на сьогодні" → впорядкований список.
- **Редагування задачі** — спливаюче вікно (той самий вигляд картки), відкривається з Inbox/Today.

## Технології

Next.js (App Router, TypeScript) + Tailwind CSS (зручно для великих touch-friendly елементів і
адаптивності), деплой на Vercel.

## Етапи

1. **Фундамент** — проєкт Next.js на Vercel, серверний виклик Anthropic API перевірений.
2. **Наскрізний сценарій** — Capture → AI розбір → Preview картки → підтвердження → localStorage →
   Inbox список (з галочкою "виконано").
3. **Today** — робочі години + кнопка "Сформувати план на сьогодні" + AI-логіка вибору/впорядкування.
4. **Онбординг** — welcome-екран, приклад у Capture, мікропідказки, порожні стани.
5. **Пізніше, не MVP** — акаунти, серверна БД і синхронізація між пристроями, нагадування, повторювані задачі.
